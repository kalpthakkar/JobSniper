Why it doesn't affect runtime:
   • Tesla adapter uses: fetch() + res.json() in JavaScript
   • Playwright's page.evaluate() returns JSON-parsed object
   • JSON parser AUTOMATICALLY decodes escape sequences
   • Python receives clean HTML without backslashes

Data flow:
   JSON in API response:    {"desc": "text<\/li>"}
   After JSON.parse():      {desc: "text</li>"}
   Python receives:         "text</li>"
   NOT:                     "text<\/li>"

2. ENHANCEMENT ADDED: JSON Escape Handling
================================================================================

Added as Step 1 in parse_html_description():

   raw_html = raw_html.replace(r'\"', '"').replace(r'\/', '/')

Why:
   • Defensive coding for unexpected cases
   • Ensures proper tag parsing if data somehow arrives JSON-escaped
   • Zero performance cost (simple string replacements)

Example:
   Before enhancement: "<\/li>" → BeautifulSoup treats \ as text
   After enhancement:  "<\/li>" → converted to "</li>" → proper parsing
